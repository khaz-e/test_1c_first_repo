# Асинхронность 1С: `Обещание`, `Асинх`, `Ждать` — правила для ИИ-агента

## 1. Назначение документа

Документ задаёт **правила и синтаксис** работы с механизмом **`Обещание` + `Асинх` + `Ждать`** во встроенном языке 1С:Предприятие. Используется как **единый справочник** при генерации и ревью кода (в том числе в сценариях управляемых форм).

**Официальная база:** описание механизма и эволюция API привязаны к версиям платформы; перед сдачей кода сверяйте сигнатуры с **ITS / синтакс-помощником** для целевой версии конфигурации.

---

## 2. Правила для ИИ-агента: когда вообще говорить про `Асинх` / `Ждать`

1. **Не предлагать `Асинх` / `Ждать` «по умолчанию» и не «улучшать» код без запроса.** Сначала решение должно быть корректным в терминах бизнес-логики, клиент/сервер и ограничений конфигурации.
2. **Рекомендация использовать `Асинх` / `Ждать` допускается только:**
   - **в конце** выполнения задачи (отдельный явный блок «дополнительно»), **или**
   - **в плане решения** задачи — отдельным пунктом с **конкретным перечислением преимуществ** для *этой* задачи (например: запрет синхронных вызовов на клиенте, цепочка файловых операций без дробления на колбэки, обязательная линейная обработка результата HTTP и т.п.).
3. Если **нет устойчивого преимущества** для конкретной задачи — **не предлагать** использование **`Асинх`/`Ждать`** в ответе и не включать его в основной план как обязательный шаг.
4. **Не подменять** типовые паттерны БСП/конфигурации собственной «переписанной асинхронностью» без явного задания пользователя и без анализа совместимости.

## 3. Версии платформы: что от чего зависит

| Версия (ориентир) | Что появилось / важно |
|--------------------|------------------------|
| **8.3.18+** | Базовый синтаксис: тип **`Обещание`**, модификатор **`Асинх`**, оператор **`Ждать`**, набор методов платформы **`*Асинх`** / **`*Async`** (возвращают `Обещание`). |
| **8.3.18 … 8.3.20** (практика проектов) | Расширение набора `*Асинх` по релизам; точный список методов — только из справки **вашей** минорной версии. |
| **8.3.21+** | Дополнительно асинхронные методы для **HTTP** (см. п. 12.4). |
| Конфигурации на **более старых** платформах | Синтаксиса `Асинх`/`Ждать` **нет**; используются **`ОписаниеОповещения`** и процедуры **`Начать…` / `Показать…`**. |

**Важно:** сама **модель** асинхронности на клиенте (веб-клиент, ограничения синхронных вызовов) существовала **до** 8.3.18; с 8.3.18 изменилась в основном **форма записи** кода (линейный стиль вместо цепочек оповещений).

**Сервер:** универсальные правила «всё на сервере пишем через `Асинх`» **неверны**. Набор допустимых асинхронных вызовов и контекстов выполнения на сервере **жёстко зависят от версии** и от типа модуля — каждый спорный вызов проверять в **документации к релизу**. Параллельный доступ к данным БД, транзакции и **`БлокировкаДанных`** описаны отдельно в **`docs/blocking_data_1c.md`** — не подменять ими асинхронный UI и наоборот.

## 4. Три составляющие механизма

1. **`Обещание`** — значение-обёртка над результатом асинхронной операции (когда-то ещё неизвестным).
2. **Методы платформы `*Асинх`** — возвращают **`Обещание`** (без параметра **`ОписаниеОповещения`** в типичном новом API).
3. **`Асинх`** на процедурах/функциях модуля и **`Ждать`** внутри них — связка «приостановить текущую процедуру до завершения `Обещания` и продолжить с той же строки».

## 5. Состояния `Обещание` и ограничения API

Логически (терминология из материалов 1С):

| Состояние | Смысл |
|-----------|--------|
| Ожидание (pending) | Операция не завершена; стартовое состояние. |
| Успех | Внутри — результат, который вернула бы операция при нормальном завершении. |
| Провал | Внутри — исключение, не перехваченное внутри асинхронной операции. |

После финала (**успех** или **провал**) состояние **не меняется**.

**Ограничения (типично для встроенного языка):**

- У **`Обещание`** нет конструктора «из ниоткуда» для прикладного кода: объект появляется как результат **`Асинх` функции платформы** (или как часть платформенного API).
- Разворачивание результата в обычное значение или проброс ошибки — через **`Ждать`**.

## 6. Синтаксис `Асинх` и `Ждать`

### 6.1. Объявление

```bsl
// Процедура с модификатором Асинх — для сценариев, где не нужно возвращать Обещание вызывающему BSL-коду.
&НаКлиенте
Асинх Процедура ОбработчикКоманды(Команда)
	// ...
КонецПроцедуры

// Функция с модификатором Асинх ВСЕГДА возвращает Обещание (см. п. 8).
&НаКлиенте
Асинх Функция ПолучитьДанныеАсинх()
	// ...
КонецФункции
```

### 6.2. Оператор `Ждать`

- **`Ждать <ВыражениеТипаОбещание>`** разрешён **только** внутри **`Асинх`** процедур и функций.
- При **успехе** выражение даёт «развёрнутое» значение.
- При **ошибке** из **`Ждать`** вылетает **то же исключение**, которое попало в `Обещание`.

```bsl
&НаКлиенте
Асинх Процедура ПримерЖдать()
	// Допустимо: сначала получить Обещание, затем ждать.
	// Состав параметров ВопросАсинх — по синтакс-помощнику целевой версии платформы.
	Обещание = ВопросАсинх(
		НСтр("ru = 'Продолжить?'"),
		РежимДиалогаВопрос.ДаНет,
		,
		КодВозвратаДиалога.Да);
	Ответ = Ждать Обещание;

	// Допустимо: короче — сразу Ждать результат вызова *Асинх.
	Ответ2 = Ждать ВопросАсинх(
		НСтр("ru = 'Удалить?'"),
		РежимДиалогаВопрос.ДаНет,
		,
		КодВозвратаДиалога.Нет);
КонецПроцедуры
```

### 6.3. Вызов `*Асинх` вне `Асинх`-процедуры

**Недопустимо:** вызвать `КакойТоМетодАсинх(...)` в обычной процедуре и написать **`Ждать`** там же — компилятор это запретит. Обернуть в **`Асинх`** процедуру/функцию или вызывать из уже **`Асинх`** контекста.

## 7. Старый и новый стиль платформы (идея миграции)

| Задача | Синхронно (где разрешено) | Старый асинхронный стиль | Новый стиль |
|--------|---------------------------|---------------------------|-------------|
| Копирование файла (клиент) | `КопироватьФайл(...)` | `НачатьКопированиеФайла(ОписаниеОповещения, ...)` | `КопироватьФайлАсинх(...)` → `Обещание`, далее **`Ждать`** |
| Вопрос пользователю | `Вопрос(...)` (модально, если политика позволяет) | `ПоказатьВопрос(ОписаниеОповещения, ...)` | `ВопросАсинх(...)` + **`Ждать`** |

**Именование:** у многих методов тройка «синхронный / с оповещением / `*Асинх`»; суффикс **`Асинх`** / **`Async`** у нового API — ориентир, не замена справке.

## 8. `Асинх` **функция** всегда возвращает `Обещание`

- Вызов **`Асинх` функции** из **обычной** (не `Асинх`) процедуры **не** выбрасывает исключения «сразу» в классическом виде: функция возвращает **`Обещание`**, а исключение проявится только после **`Ждать`**.
- Чтобы перехватить ошибку **`Попытка`/`Исключение`**, оборачивать нужно именно **`Ждать`**, а не «голый» вызов **`Асинх` функции** без ожидания.

```bsl
// Неверно: исключение из ПолучитьОбещаниеСОшибкой() не будет перехвачено здесь.
&НаКлиенте
Процедура ПлохойПерехват()
	Попытка
		// Вызов без Ждать: возвращается Обещание, исключение из асинхронной части сюда не попадёт.
		НеиспользуемоеОбещание = ПолучитьОбещаниеСОшибкой();
	Исключение
		// сюда не попадём из-за Обещания
	КонецПопытки;
КонецПроцедуры

&НаКлиенте
Асинх Функция ПолучитьОбещаниеСОшибкой()
	ВызватьИсключение "ошибка внутри";
КонецФункции

// Верно: перехват вокруг Ждать внутри Асинх-процедуры.
&НаКлиенте
Асинх Процедура ХорошийПерехват()
	Попытка
		Ждать ПолучитьОбещаниеСОшибкой();
	Исключение
		Сообщить(ОписаниеОшибки());
	КонецПопытки;
КонецПроцедуры
```

## 9. `Асинх` **процедура**: исключения и вызывающий код

- **`Асинх` процедура** значения **не возвращает**.
- Неперехваченное исключение в **`Асинх` процедуре** обычно приводит к **стандартному отображению ошибки**; обернуть вызов такой процедуры **`Попытка`/`Исключение`** «снаружи» **нельзя** так же, как для синхронного кода.
- Если нужно **программно** обработать сбой в цепочке — проектируйте **`Асинх` функцию** и **`Ждать`** её результат в вызывающей **`Асинх`** процедуре.

**Практический вывод:** для «командной кнопки» **`Асинх` процедура** часто удобна; для **библиотечной** операции с контролем ошибок — чаще **`Асинх` функция**.

---

## 10. Параметры `Асинх` процедур и функций — только по значению

В **`Асинх`**-контексте параметры передаются **по значению**; наличие или отсутствие ключевого слова **`Знач`** в объявлении **не меняет** эту модель.

**Критично для форм:** параметр **`Отказ`** в обработчиках событий, объявленный как **`Асинх Процедура …(Отказ)`**, **не позволит** отменить действие присвоением **`Отказ = Истина`** — это та же логическая ошибка, что и у **`Процедура …(Знач Отказ)`** в обычном коде: изменяется **копия**, а не параметр платформы.

**Правило для форм:** обработчики **`ПриОткрытии`**, **`ПередЗакрытием`** и др., где нужны **`Отказ`**, стандартная обработка и синхронизация с платформой, — оформлять как **обычные** процедуры; асинхронный вопрос выносить во **внутреннюю** **`Асинх`** процедуру **без** параметра **`Отказ`** в сигнатуре `Асинх`-обёртки (см. п. 11).

## 11. Формы: вопрос при закрытии (`ВопросАсинх` + флаг)

Идея: в **`ПередЗакрытием`** один раз отменяем стандартное закрытие (**`Отказ = Истина`**), запускаем **`Асинх`**-процедуру с **`Ждать ВопросАсинх`**; после подтверждения вызываем **`Закрыть()`**; флаг не даёт зациклить вопрос.

```bsl
&НаКлиенте
Перем ВопросПриЗакрытииЗадан;

// Инициализация — в клиентской процедуре ПриОткрытии: ВопросПриЗакрытииЗадан = Ложь;

&НаКлиенте
Процедура ПередЗакрытием(Отказ, ЗавершениеРаботы, ТекстПредупреждения, СтандартнаяОбработка)

	Если НЕ ВопросПриЗакрытииЗадан Тогда
		ЗадатьВопросПриЗакрытииАсинх();
		Отказ = Истина; // закрытие отложено до ответа пользователя
	КонецЕсли;

КонецПроцедуры

&НаКлиенте
Асинх Процедура ЗадатьВопросПриЗакрытииАсинх()

	Обещание = ВопросАсинх(
		НСтр("ru = 'Закрыть форму?'"),
		РежимДиалогаВопрос.ДаНет,
		,
		КодВозвратаДиалога.Да);
	Результат = Ждать Обещание;

	Если Результат = КодВозвратаДиалога.Да Тогда
		ЭтаФорма.Закрыть();
	КонецЕсли;

	ВопросПриЗакрытииЗадан = Истина;

КонецПроцедуры
```

**Важно:** состав параметров **`ПередЗакрытием`** зависит от версии платформы; при копировании в конфигурацию приведите сигнатуру к **синтакс-помощнику**.

## 12. Примеры по типам задач

### 12.1. Диалоги: `ПредупреждениеАсинх` с `Ждать` и без

С **`Ждать`** — выполнение **останавливается** до закрытия окна. Без **`Ждать`** — окно показано, а код **ниже выполнится сразу**.

```bsl
&НаКлиенте
Асинх Процедура ДемоПредупреждение()

	Ждать ПредупреждениеАсинх(НСтр("ru = 'Сначала это'"));

	// Второе предупреждение без ожидания — дальнейший код не ждёт закрытия окна.
	ПредупреждениеАсинх(НСтр("ru = 'Второе (не ждём)'"));

	Сообщение = Новый СообщениеПользователю;
	Сообщение.Текст = НСтр("ru = 'Это сообщение может появиться до закрытия второго предупреждения'");
	Сообщение.Сообщить();

КонецПроцедуры
```

### 12.2. Что возвращает `Ждать` для частых методов (ориентир; сверять с справкой)

| Метод (пример) | Типичный результат `Ждать` при успехе |
|----------------|--------------------------------------|
| `ПредупреждениеАсинх` | `Неопределено` |
| `ВопросАсинх` | `КодВозвратаДиалога` |
| `ОткрытьЗначениеАсинх` | `Неопределено` |
| `ВвестиЧислоАсинх` | введённое **число** |
| `НайтиФайлыАсинх` | **массив** `Файл` |

Если у старого API в оповещении не было «полезного» результата, при успехе в **`Обещании`** может быть **`Неопределено`**.

### 12.3. Помещение файла на сервер через диалог: `ПоместитьФайлНаСерверАсинх`

С **8.3.18** доступен **`ПоместитьФайлНаСерверАсинх`**. Типичный паттерн — передать **`ПараметрыДиалогаПомещенияФайлов`** и идентификатор владельца ВХ (например, **`ЭтаФорма.УникальныйИдентификатор`**).

```bsl
&НаКлиенте
Асинх Процедура ЗагрузитьФайлЧерезДиалог(Команда)

	ПараметрыДиалога = Новый ПараметрыДиалогаПомещенияФайлов(
		НСтр("ru = 'Выберите файл'"),
		Ложь,
		"Картинки (*.jpg)|*.jpg",
		0);

	Обещание = ПоместитьФайлНаСерверАсинх(,,, ПараметрыДиалога, ЭтаФорма.УникальныйИдентификатор);
	Результат = Ждать Обещание;

	Если Результат = Неопределено Тогда
		Возврат; // пользователь отменил или ошибка — уточнять по документации
	КонецЕсли;

	ОбработатьАдресНаСервере(Результат.Адрес);

КонецПроцедуры

&НаСервереБезКонтекста
Процедура ОбработатьАдресНаСервере(Знач АдресВременногоХранилища)
	ДвоичныеДанные = ПолучитьИзВременногоХранилища(АдресВременногоХранилища);
	// ...
КонецПроцедуры
```

**Замечание:** конструктор **`ПараметрыДиалогаПомещенияФайлов`** и состав полей результата **`Результат`** зависят от версии — **проверка в справке обязательна**.

Альтернатива — связка **`ДиалогВыбораФайлов`** + **`Ждать Диалог.ВыбратьАсинх()`** (если метод доступен в вашей версии) и далее помещение файла другим API.

### 12.4. HTTP (`*Асинх`) с **8.3.21+** (перечень из публикаций; детали — в справке)

- `ВызватьHTTPМетодАсинх`
- `ЗаписатьАсинх`, `ИзменитьАсинх`, `УдалитьАсинх`
- `ПолучитьАсинх`, `ПолучитьЗаголовкиАсинх`
- `ОтправитьДляОбработкиАсинх`

## 13. Модель выполнения: что означает `Ждать` для UI

**`Ждать` не означает** «заблокировать весь интерфейс ОС в стиле синхронного модального вызова намертво» в абстрактном смысле: это **платформенная** приостановка **`Асинх`**-процедуры с сохранением стека и локальных переменных до готовности **`Обещания`** (подробнее в статье 1С:Зазеркалье).

На практике отдельные диалоги **могут** ограничивать ввод в форме, пока открыты; это **не отменяет** необходимости **`Асинх`/`Ждать`** там, где синхронные вызовы запрещены политикой приложения.
