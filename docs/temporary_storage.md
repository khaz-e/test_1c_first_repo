# Временное хранилище в 1С: правила для ИИ-агента

## 1. Назначение документа

Документ задает практические правила использования временного хранилища (ВХ) при обмене данными между клиентом и сервером в управляемых формах.

Для синтаксиса **`Асинх`**, оператора **`Ждать`**, типа **`Обещание`** и методов платформы **`*Асинх`** (в т.ч. помещение файла на сервер с ожиданием) используйте **`docs/async_await_1c.md`**. Если фоновая или серверная обработка по адресу ВХ ведёт к **записи в БД** при возможной конкуренции сеансов — дополнительно **`docs/blocking_data_1c.md`**. Тяжёлые **выборки** из БД на языке запросов 1С в таких сценариях — **`docs/query_language_audit_1c.md`**.

## 2. Что такое ВХ и когда оно нужно

1. **Определение.** Временное хранилище — серверная область памяти для временного размещения данных.
2. **Типовые задачи.**
   - передача файлов и двоичных данных;
   - передача больших структур данных;
   - передача данных в асинхронных/фоновых сценариях.
3. **Паттерн «ссылка вместо объекта».**
   - между клиентом и сервером передается адрес (`e1cib/tempstorage/...`);
   - реальный объект извлекается уже на нужной стороне вызова.

## 3. Критичное правило управления памятью

При вызовах `ПоместитьВоВременноеХранилище` (и родственных механизмов) всегда задавать владельца/идентификатор времени жизни данных (например, `ЭтаФорма.УникальныйИдентификатор`), чтобы исключить утечки RAM на сервере.

## 4. Шаблоны использования

### 4.1 Загрузка файла с клиента на сервер (асинхронно, 8.3.18+)

Оператор **`Ждать`** и вызовы **`*Асинх`** допустимы **только** внутри процедур/функций с модификатором **`Асинх`**. Имена параметров конструктора **`ПараметрыДиалогаПомещенияФайлов`** и поля результата **`ПоместитьФайлНаСерверАсинх`** сверяйте со справкой; альтернативы (диалог **`ВыбратьАсинх`**, обработка ошибок) — в **`docs/async_await_1c.md`**.

```bsl
&НаКлиенте
Асинх Процедура ЗагрузитьФайлИВызватьСервер()

	ПараметрыДиалога = Новый ПараметрыДиалогаПомещенияФайлов(
		НСтр("ru = 'Выберите файл'"),
		Ложь,
		"Картинки (*.jpg)|*.jpg",
		0);

	Обещание = ПоместитьФайлНаСерверАсинх(,,, ПараметрыДиалога, ЭтаФорма.УникальныйИдентификатор);
	Результат = Ждать Обещание;

	Если Результат = Неопределено Тогда
		Возврат;
	КонецЕсли;

	ОбработатьФайлНаСервере(Результат.Адрес);

КонецПроцедуры

&НаСервереБезКонтекста
Процедура ОбработатьФайлНаСервере(Знач АдресХранилища)
	ДвоичныеДанныеФайла = ПолучитьИзВременногоХранилища(АдресХранилища);
	// Дальнейшая обработка файла
КонецПроцедуры
```

### 4.2 Передача ТаблицыЗначений во внеконтекстный вызов

Подготовка ТЗ на сервере в контексте формы (**`&НаСервере`**); адрес ВХ возвращается **функцией** — так клиент получает строковый адрес для последующего **`&НаСервереБезКонтекста`**.

```bsl
&НаСервере
Функция ПодготовитьДанныеНаСервере()

	ТЗ = Новый ТаблицаЗначений;
	ТЗ.Колонки.Добавить("Товар");
	ТЗ.Колонки.Добавить("Количество");

	НоваяСтрока = ТЗ.Добавить();
	НоваяСтрока.Товар = Справочники.Номенклатура.НайтиПоНаименованию("Дрель");
	НоваяСтрока.Количество = 10;

	Возврат ПоместитьВоВременноеХранилище(ТЗ, ЭтаФорма.УникальныйИдентификатор);

КонецФункции

&НаКлиенте
Процедура ВыполнитьРасчет(Команда)

	АдресДанных = ПодготовитьДанныеНаСервере();
	РезультатРасчета = БыстрыйРасчетНаСервере(АдресДанных);

КонецПроцедуры

&НаСервереБезКонтекста
Функция БыстрыйРасчетНаСервере(Знач АдресДанных)

	ТаблицаДляРасчета = ПолучитьИзВременногоХранилища(АдресДанных);
	Возврат "Расчет окончен, обработано строк: " + ТаблицаДляРасчета.Количество();

КонецФункции
```

### 4.3 Фоновая обработка через ВХ

1. Клиент создает `УникальныйИдентификатор`/адрес ВХ.
2. Клиент запускает фоновое задание и передает этот адрес.
3. Фоновое задание кладет результат в ВХ:
   - `ПоместитьВоВременноеХранилище(Результат, АдресВХ);`
4. Клиент по завершении фонового задания извлекает результат:
   - `ПолучитьИзВременногоХранилища(АдресВХ)`.

## 5. Самопроверка перед выдачей кода

1. Для клиентских файловых операций при запрете синхронных вызовов используется валидный синтаксис **`Асинх`/`Ждать`** и методов **`*Асинх`** (см. **`docs/async_await_1c.md`**)?
2. Во всех операциях помещения в ВХ задан второй параметр (владелец/адрес)?
3. Нет ли прямой передачи тяжелых данных там, где логичнее передавать адрес ВХ?
4. Корректно ли разделены клиентская и серверная ответственность?
